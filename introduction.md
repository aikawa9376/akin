# 関連ファイル探索CLIツール「akin」開発要件
## プロジェクト概要
指定したファイルに対して、プロジェクト内から関連するファイル（テストファイルや、MVCアーキテクチャにおけるControllerに対するViewなど）を自動で探索し、リストアップするCLIツールを開発する。
単純なルールベース（正規表現やパスのハードコード）によるマッピングではなく、ファイルパスおよびファイル名の文字列の「類似度（近似値）」を厳密に計算することで、プロジェクト構造の変更に強い柔軟な探索を実現する。
## 基本情報
 * **コマンド名:** akin
 * **開発言語:** Rust
 * **主要なユースケース:** akin path/to/target_file.ext を実行し、類似度スコアの高い関連ファイルのパスを標準出力にリストアップする。
## コアアルゴリズムとアプローチ
文字列の類似度計算を用いて関連ファイルを特定する。ディレクトリ階層の違いを吸収するため、単一のアルゴリズムだけでなく前処理を組み合わせる。
### 1. 推奨される類似度計算手法
 * **レーベンシュタイン距離（編集距離）:** プレフィックスやサフィックスの差異（例: Target と TargetTest）の評価に活用。
 * **ジャロ・ウィンクラー距離:** 先頭の命名規則が共通している浅い階層の比較に活用。
 * **N-gram + Jaccard係数（トークンベース比較）:** パスを単語ごとに分割し、階層が全く異なるが共通ドメイン名を持つファイル（例: controllers/user と views/user）の発見に活用。
### 2. 探索の精度を上げるための前処理
類似度計算の前に、パス文字列に対して以下の正規化処理を行う。
 * **拡張子の除去:** .php、.rs、.ts などの拡張子を計算対象から外す。
 * **小文字化:** 比較のブレを防ぐため、すべて小文字（lowercase）に統一する。
 * **トークン化（単語分割）:** パスをディレクトリセパレータ（/ や \）、およびキャメルケースやスネークケースの区切りで分割する。
### 3. スコアの重み付け（ヒューリスティック）
 * ファイルパスの末尾（ファイル名本体）に共通のトークンが含まれている場合は、スコアに高い重みを付ける。
 * 例外的なディレクトリ名のノイズ（app、src、resources など）に判定が引っ張られないようにトークンの重みを調整する。
## 処理フロー
 1. CLI引数からターゲットファイルのパスを受け取る。
 2. プロジェクト内のファイル一覧を取得する（.gitignore に含まれるファイルやディレクトリは除外する）。
 3. 取得した全ファイルパスと、ターゲットファイルのパスに対して前処理（正規化・トークン化）を実行する。
 4. ターゲットファイルと各ファイルの間で類似度スコアを計算する。
 5. スコアが一定の閾値を超えるもの、またはスコア上位N件をソートして結果を出力する。
## 利用を想定しているRustクレート
 * **clap**: CLIの引数解析用。
 * **ignore**: ディレクトリ内のファイル再帰検索および .gitignore の自動解釈用（探索パフォーマンス向上とノイズ削減の要）。
 * **strsim**: レーベンシュタイン距離やジャロ・ウィンクラー距離などの一般的な文字列類似度計算用。
